### 設定 CI/CD

建立 `.github/workflows/test.yml`：

```yaml
name: Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.10'
      - run: pip install -r requirements.txt
      - run: pytest tests/ -v
      - run: python benchmark.py --quick  # 10 分鐘快速驗證
```

## 🎓 你學到了什麼

### 技術經驗

1. **論文超參數不能轉移**
   - 原始 Transformer 排程在小型模型上失敗
   - 簡單的固定學習率效果更好
   - 總是在你的特定設定上驗證

2. **在記錄之前測試聲明**
   - 我們最初在未測試的情況下聲稱反轉/排序有效
   - 如果公開發布可能會很尷尬
   - 基準測試自動驗證聲明

3. **Jupyter 不總是答案**
   - 專業程式碼 > notebook 探索
   - 自動化測試 > 手動驗證
   - 版本控制很重要

### 流程經驗

1. **從最小驗證開始**
   - 過擬合測試證明架構有效
   - 單一任務證明參數有效
   - 完整基準測試驗證所有內容

2. **隨時記錄**
   - 建立全面指南
   - 未來使用者不會重複錯誤
   - 保存知識

3. **自動化防止回歸**
   - 基準測試捕捉破壞性變更
   - CI/CD 強制執行品質
   - 測試就是文件
